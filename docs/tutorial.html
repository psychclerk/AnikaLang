LESSON 1

Setup & Your First Program

Get AnikaLang running and write your very first script in under 2 minutes.

📦 Installation

AnikaLang runs on top of wxPython. Install it with one command:

Terminal
1pip install wxPython

📝 Your First Script

Create a file called hello.fms and add this code:

hello.fms
1# This is a comment - AnikaLang ignores it
2SET win TO UI_WINDOW("Hello World", 400, 300)
3SET lbl TO UI_LABEL(win, "Hello, World!", 150, 130, 100, 30)
4UI_MAINLOOP(win)

Run it from your terminal:

Terminal
1python anikawx.py hello.fms

You should see a window with "Hello, World!" in the middle. 🎉

🔍 What Just Happened?

  • UI_WINDOW creates a window with a title, width, and height
  • UI_LABEL creates a text label at position (150, 130) with size (100, 30)
  • UI_MAINLOOP starts the event loop — without this, the window closes immediately
💡
Pro Tip
Every AnikaLang GUI app ends with UI_MAINLOOP(win). Think of it as "keep the window open until the user closes it."

Variables & Data Types

Store values, check types, and convert between data types.

📦 Creating Variables

Use SET name TO value to create variables:

1SET name TO "Alice" # String
2SET age TO 30 # Integer
3SET height TO 5.7 # Float
4SET active TO TRUE # Boolean
5SET nothing TO NULL # Null value

🔍 Type Checking & Conversion

types.fms
1# Check the type of a value
2SET t TO TYPE_OF(age) # Returns "INTEGER"
3
4# Convert between types
5SET num_str TO "42"
6SET num TO INT(num_str) # 42
7SET pi_str TO STR(3.14159) # "3.14159"
8SET price TO FLOAT("19.99") # 19.99

📝 String Operations

strings.fms
1SET greeting TO "Hello" & " " & "World" # Concatenation
2SET upper TO UPPER("hello") # "HELLO"
3SET lower TO LOWER("HELLO") # "hello"
4SET length TO LEN("hello") # 5
5SET part TO MID("hello", 1, 3) # "ell"
ℹ️
String Concatenation
Use the & operator to join strings together. The + operator is for math only.
LESSON 3

Operators & Expressions

Math, comparisons, and logical operations.

🧮 Math Operators

math.fms
1SET a TO 10 + 5 # 15 (addition)
2SET b TO 10 - 5 # 5 (subtraction)
3SET c TO 10 * 5 # 50 (multiplication)
4SET d TO 10 / 3 # 3.333... (division)
5SET e TO POW(2, 10) # 1024 (power)
6SET f TO SQRT(16) # 4 (square root)
⚠️
No Modulo or Power Operators
AnikaLang doesn't have % or ^ operators. Use POW(base, exp) for power. For modulo, use: a - INT(a/b)*b

⚖️ Comparison & Logic

logic.fms
1SET age TO 25
2SET has_id TO TRUE
3
4# AND - both must be true
5IF age >= 18 AND has_id == TRUE THEN
6 UI_ALERT("Can enter")
7END IF
8
9# OR - at least one must be true
10IF age < 13 OR age > 65 THEN
11 UI_ALERT("Discount available")
12END IF
OperatorMeaningExample
==Equal5 == 5 → TRUE
!=Not equal5 != 3 → TRUE
<, >Less / Greater3 < 5 → TRUE
<=, >=Less-equal / Greater-equal5 >= 5 → TRUE
ANDLogical ANDTRUE AND FALSE → FALSE
ORLogical ORTRUE OR FALSE → TRUE
NOTLogical NOTNOT TRUE → FALSE

Control Flow

Make decisions with IF/ELSE and repeat actions with loops.

🔀 IF / ELSE Decisions

ifelse.fms
1SET score TO 85
2
3IF score >= 90 THEN
4 SET grade TO "A"

6 IF score >= 80 THEN
7 SET grade TO "B"
8 ELSE
9 SET grade TO "C"
10 END IF

AnikaLang doesn't have ELSE IF. To handle multiple conditions, nest IF inside ELSE as shown above.

Lists & Dictionaries

Store collections of data with lists and key-value pairs with dictionaries.

📋 Lists

lists.fms
1# Create a list
2SET fruits TO ["apple", "banana", "cherry"]
3
4# Access by index (0-based)
5SET first TO fruits[0] # "apple"
6SET last TO fruits[2] # "cherry"
7
8# Modify the list
9LIST_SET(fruits, 1, "blueberry") # Replace index 1
10LIST_APPEND(fruits, "date") # Add to end
11
12# Length
13SET count TO LEN(fruits) # 4

🗂️ Dictionaries

dicts.fms
1# Create a dictionary
2SET person TO {
3 "name": "Alice",
4 "age": 30,
5 "city": "NYC"
6}
7
8# Access values
9SET name TO person["name"] # "Alice"
10SET name TO person.name # Same thing!
11
12# Modify
13DICT_SET(person, "age", 31)
14DICT_SET(person, "email", "alice@example.com")
15
16# Check if key exists
17IF DICT_HAS_KEY(person, "email") THEN
18 UI_ALERT("Has email")
19END IF
LESSON 7

Your First GUI App

Build a complete counter application with buttons and a label.

Let's combine everything you've learned into a real interactive app:

counter.fms
1# Counter App
2SET count TO 0
3SET counter_lbl TO NULL
4
5FUNCTION increment()
6 SET count TO count + 1
7 UI_SET(counter_lbl, "Count: " & STR(count))
8END FUNCTION
9
10FUNCTION reset()
11 SET count TO 0
12 UI_SET(counter_lbl, "Count: 0")
13END FUNCTION
14
15# Create window
16SET win TO UI_WINDOW("Counter App", 300, 200)
17
18# Create label
19SET counter_lbl TO UI_LABEL(win, "Count: 0", 100, 50, 100, 30)
20
21# Create buttons
22SET btn_inc TO UI_BUTTON(win, "Increment", "increment", 50, 100, 90, 30)
23SET btn_reset TO UI_BUTTON(win, "Reset", "reset", 160, 100, 90, 30)
24
25# Start event loop
26UI_MAINLOOP(win)

🔍 How It Works

  1. We define two functions: increment and reset
  2. We create a window, label, and two buttons
  3. Each button is linked to a function by name as a string
  4. When clicked, the button calls the function
  5. The function updates the label using UI_SET
💡
Global Variables
Notice count and counter_lbl are declared at the top. Functions can access and modify global variables directly.

7END FUNCTION

9UI_MAINLOOP(win)

Use UI_BIND for events other than clicks:

bind.fms
1SET win TO UI_WINDOW("Event Binding", 400, 200)

5# Bind CHANGE event (fires when text changes)
6UI_BIND(ent, "CHANGE", "on_change")
7
8SET lbl TO UI_LABEL(win, "Type something...", 20, 60, 360, 25)

12 UI_SET(lbl, "You typed: " & text)
13END FUNCTION

All widgets use coordinates (x, y, width, height):

📏 Responsive Resize

Bind the RESIZE event to make your app adapt to window size:

resize.fms
1SET win TO UI_WINDOW("Responsive", 800, 600)
2
3SET editor TO UI_TEXT(win, "", TRUE, 0, 0, 600, 570)
4SET statusbar TO UI_PANEL(win, 0, 570, 800, 30)
5
6UI_BIND(win, "RESIZE", "on_resize")
7
8FUNCTION on_resize(widget, w, h)
9 # Get actual drawable area
10 SET cs TO UI_GET_CLIENT_SIZE(win)
11 SET cw TO cs[0]
12 SET ch TO cs[1]
13
14 # Resize editor to fill available space
15 UI_SIZE(editor, cw, ch - 30)
16
17 # Move status bar to bottom
18 UI_POS(statusbar, 0, ch - 30, cw, 30)
19END FUNCTION
20
21UI_MAINLOOP(win)
💡
Use UI_GET_CLIENT_SIZE
Always use UI_GET_CLIENT_SIZE(win) instead of UI_GET_SIZE(win) for layout calculations. It returns the drawable area, excluding title bar and menu bar.

Build a Complete App

Put it all together with a full-featured Todo List application.

Let's build a complete Todo List app that uses everything you've learned:

📋 Todo List Application

A fully functional app with add, toggle, delete, and save features.

todo.fms — Complete App
1# ==============================================================================
2# TODO LIST APPLICATION
3# ==============================================================================
4
5# Global state
6SET todos TO []
7SET todo_list TO NULL
8SET entry TO NULL
9
10# --- Functions ---
11
12FUNCTION add_todo()
13 SET text TO UI_GET(entry)
14 IF LEN(TRIM(text)) == 0 THEN
15 UI_ALERT_ERR("Please enter a todo item")
16 RETURN
17 END IF
18
19 # Add to list
20 SET new_todo TO {"text": text, "done": FALSE}
21 LIST_APPEND(todos, new_todo)
22
23 # Clear entry
24 UI_SET(entry, "")
25
26 # Refresh display
27 refresh_list()
28END FUNCTION
29
30FUNCTION toggle_todo()
31 SET selected TO UI_LISTVIEW_GET_SELECTED(todo_list)
32 IF selected == NULL THEN
33 UI_ALERT_ERR("Please select a todo")
34 RETURN
35 END IF
36
37 # Find the todo by text
38 SET row_text TO selected[1]
39 FOR todo IN todos THEN
40 IF todo["text"] == row_text THEN
41 IF todo["done"] == TRUE THEN
42 DICT_SET(todo, "done", FALSE)
43 ELSE
44 DICT_SET(todo, "done", TRUE)
45 END IF
46 BREAK
47 END IF
48 END FOR
49
50 refresh_list()
51END FUNCTION
52
53FUNCTION refresh_list()
54 UI_LISTVIEW_CLEAR(todo_list)
55 FOR todo IN todos THEN
56 SET status TO "⬜"
57 IF todo["done"] == TRUE THEN
58 SET status TO "✅"
59 END IF
60 UI_LISTVIEW_INSERT(todo_list, [status, todo["text"]])
61 END FOR
62END FUNCTION
63
64FUNCTION save_todos()
65 SET json TO JSON_STRINGIFY(todos)
66 FILE_WRITE("todos.json", json)
67 UI_ALERT("Todos saved to todos.json")
68END FUNCTION
69
70# --- UI Layout ---
71SET win TO UI_WINDOW("Todo List", 500, 400)
72
73# Title
74SET title TO UI_LABEL(win, "📝 Todo List", 200, 10, 100, 30)
75UI_FONT(title, 16, "Segoe UI", TRUE)
76
77# Entry field
78SET entry TO UI_ENTRY(win, "Enter todo...", 20, 50, 360, 25)
79
80# Buttons
81SET btn_add TO UI_BUTTON(win, "Add", "add_todo", 390, 50, 90, 25)
82UI_COLOR(btn_add, "white", "#27ae60")
83
84SET btn_toggle TO UI_BUTTON(win, "Toggle", "toggle_todo", 20, 90, 90, 25)
85UI_COLOR(btn_toggle, "white", "#3498db")
86
87SET btn_save TO UI_BUTTON(win, "Save", "save_todos", 120, 90, 90, 25)
88UI_COLOR(btn_save, "white", "#9b59b6")
89
90# ListView
91SET todo_list TO UI_LISTVIEW(win, ["Status", "Todo"], 20, 130, 460, 240)
92UI_LISTVIEW_SET_COLUMN_WIDTH(todo_list, 0, 60)
93UI_LISTVIEW_SET_COLUMN_WIDTH(todo_list, 1, 380)
94
95# Start
96UI_MAINLOOP(win)

LESSON 14

Common Pitfalls

Avoid these common mistakes that trip up beginners.

❌ Pitfall 1: Using ELSE IF

AnikaLang doesn't have ELSE IF. You must nest IF inside ELSE:

# WRONG - Syntax error
IF x > 10 THEN
    ...
ELSE IF x > 5 THEN # ❌ Error!
    ...
END IF
# CORRECT - Nest IF inside ELSE
IF x > 10 THEN
    ...
ELSE
    IF x > 5 THEN # ✅ Correct
        ...
    END IF
END IF
❌ Pitfall 2: Using % or ^ Operators

AnikaLang doesn't have modulo (%) or power (^) operators:

SET x TO 10 % 3 # ❌ Syntax error
SET y TO 2 ^ 10 # ❌ Syntax error
SET y TO POW(2, 10) # ✅ Use POW function
❌ Pitfall 3: Forgetting UI_MAINLOOP

Without UI_MAINLOOP, the window closes immediately:

SET win TO UI_WINDOW("Test", 300, 200)
UI_LABEL(win, "Hello", 100, 80, 100, 30)
# Missing UI_MAINLOOP(win) — window closes immediately!
SET win TO UI_WINDOW("Test", 300, 200)
UI_LABEL(win, "Hello", 100, 80, 100, 30)
UI_MAINLOOP(win) # ✅ Keeps window open
❌ Pitfall 4: Wrong Function Name in Button

Function names in UI_BUTTON must be strings:

# ❌ on_click is a variable, not a string
SET btn TO UI_BUTTON(win, "Click", on_click, 100, 100, 100, 30)
# ✅ Function name as string
SET btn TO UI_BUTTON(win, "Click", "on_click", 100, 100, 100, 30)
❌ Pitfall 5: Modifying Lists Incorrectly

Can't use bracket assignment — use helper functions:

SET list TO [1, 2, 3]
SET list[0] TO 10 # ❌ Syntax error
SET list TO [1, 2, 3]
LIST_SET(list, 0, 10) # ✅ Use LIST_SET
❌ Pitfall 6: Using Variables Before Defining

Variables must be defined before use:

UI_ALERT(message) # ❌ message not defined yet
SET message TO "Hello"
SET message TO "Hello"
UI_ALERT(message) # ✅ Works now
